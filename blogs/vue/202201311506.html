    Vue的双向绑定原理探索

    GuanDichen 2022/1/31 Vue

    背景

    不知不觉接触 Vue 也有一段时间了，之前也解过 Vue 的响应性原理，但不够深入，毕竟能把框架用得好的，一般都是建立在懂底层原理的基础上。其实 Vue 文档也有关于这块核心的讲解，我当时看到这一块的时候，也是一头雾水，花了一两天时间，结合一些代码的例子看，才消化了个大概。

    # 响应式原理

    Vue2 官方文档 (opens new window)

    Vue 的核心主要是响应性系统和组件化，数据驱动视图，视图层更改同步到数据层。

    Vue 实际上通过Object.defineProperty把对象property转换为getter和setter来进行数据劫持，并结合发布/订阅模式 (opens new window)，来实现依赖追踪。

    核心的实现部分：
    Observer: 主要负责主题更新的通知发布。
    Watcher：作为 observer 和 compilier 沟通的桥梁。
    Compiler: 主要负责进行模板的编译和视图的更新。

    # 工作过程

    组件在初次渲染时，访问 proterty(来自data或prop)的过程中，触发依赖项的getter，被收集为该 property 的依赖Watcher，所以每个组件实例都对应一个Watcher实例，之后在组件运行过程中，如果依赖项的 property 发生变化，就会触发setter，然后通知Watcher，触发组件重新渲染。

    # 变更检测

    Vue对不同类型property的变动检测方式不同，例如：对数组元素的增、删、改操作；嵌套对象的 property 直接赋值，可能无法直接检测到，因为getter和setter，只会在property访问和重新赋值时被调用。为了能让数组的某些方法也能触发， Vue 会对数组的方法经过包裹处理。

    // 不会触发
Object.assign(this.ob, { a: 1, b: 2 });
// 触发更新
this.ob = Object.assign({}, this.ob, { a: 1, b: 2 });
