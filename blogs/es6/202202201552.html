    初识Promise

    GuanDichen 2022/2/20 JavaScript

    ES6 里新增的Promise对象，主要用来解决异步编程中的“回调地狱”问题。
    与现实生活中的类比：
    就好像你在 B 站上关注了某位 up 主后，通常，up 主有了新的视频发布，你也会第一时间收到内容更新通知。

    1. “生产者代码（Producer code）”需要一段时间来做某件事，例如，通过网络加载的数据，它就像一个 up 主。
    2. “消费者代码（comsumer code）”需要在生产者代码完成工作的第一时间收获它的成果。例如，异步编程中回调函数，它就像 B 站的普通用户。
    3. Promise是将“生产者代码”和“消费者代码”连接在一起的特殊 JavaScript 对象。它就像是“订阅列表”，“生产者代码”需要花费任意时间来产出所承诺的成果，promise 将在它准备好时，将结果向所有订阅了的代码推送。

    Promise构造函数的语法如下：

    let promise = new Promise(function(resolve, reject) {
  // executor (生产者代码，“up主”)
});
    1
    2
    3

    传递给new Promise()的函数称为executor。
    它的两个参数resolve和reject是 JavaScript 自身提供的回调函数。我们的代码在executor内部。 当executor有了成果，应该调用以下回调之一：

    • resolve(value)——任务完成并带有结果，就调用这个。
    • reject(error)——如果出现 error，就调用这个

    下面是一个 promise 构造函数和简单的 executor 函数的例子：

    let promise = new Promise((resolve, reject) => {
  // 1秒后发出工作完成后的信号，并返回一个error
  setTimeout(() => resolve("done"), 1000);
});

    let promise = new Promise((resolve, reject) => {

  setTimeout(() => reject(new Error("Timeout!")), 1000);

  setTimeout(() => resolve("..."), 1000); // 被忽略

    # 消费者：then，catch，finally

    Promsie 充当 executor（生产者代码或“up 主”）与消费函数（订阅的用户）之间的连接，后者接收成果或 error，通过then，catch，finally为消费函数进行注册。

    # then

    语法如下：

    promsie.then(
  (result) => {}, // handle a successful result
  (error) => {} // handle an error
);

    # finally

    跟try {...} catch {...}中的finally子句一样，promise 也有自己的finally
    finally适合用来执行清理操作（handle），例如不管结果如何，都停止加载指示器（indicator）。 像这样：

    new Promise((resolve, reject) => {
  // 进行一个耗时的操作，稍后调用resolve或reject
})
  .finally(() => stopLoading()) // promise为settled时执行，无论成功与否
  .then(
    // 所以加载指示器（loading indicator）始终会在我们处理结果或错误之前停止
    (result) => alert(result),
    (err) => alert(err)
  );

    # 示例：loadScript

    loadScript该函数不需要传递回调，它将创建并返回一个在加载完成时解析（resolve）的 promise 对象，并在外部.then向其添加处理函数。

    function loadScript(src) {
  return new Promise((resolve, reject) => {
    let script = document.createElement("script");
    script.src = src;

    script.onload = () => resolve(script);
    script.onerror = () =>
      reject(new Error(`Script failed to laod for ${src}`));

    document.head.append(script);
  });
}

// 使用：加载vue3源码
let promise = loadScript(
  "https://unpkg.com/react@16.14.0/umd/react.development.js"
);

promise.then(
  (script) => alert(`${script.src} is loaded!`),
  (error) => alert(`Error: ${error}`)
);

    # promise 链

    promise 允许通过.then进行链式调用，因为每个.then中所使用的处理程序（handle）都会创建并返回一个 Promise，其他的处理程序（handle）必须等待它 settled 后再获得其结果。

    loadScript("https://unpkg.com/react@16.14.0/umd/react.development.js")
  .then((script) => loadScript("https://code.jquery.com/jquery-3.6.0.min.js"))
  .then((script) => {
    // 使用在脚本中声明的函数
    alert(`React version: ${React.version}`);
    alert($("title")[0].text);
  });

    # 隐式 try...catch

    Promise的执行者executor和promise的处理程序（handle）周围有一层“隐式的try...catch”，如果发生异常，它就会被捕获，并被视为 rejection 进行处理。
    所以.catch不仅会捕获显式的 rejection，还会捕获它上面处理程序（handle）中意外出现的 error。


     


    new Promise((resolve, reject) => {
  reject(new Error("syntax number"));
}).catch(alert);
    1
    2
    3

    与下面这段代码功能相同


     


    new Promise((resolve, reject) => {
  throw new Error("syntax number");
}).catch(alert);

    # setTimeout 中的错误

    下面代码中异常会被.catch捕获到吗？

    new Promise(function(resolve, reject) {
  setTimeout(() => {
    throw new Error("number syntax");
  }, 1000);
}).catch(alert);
    1
    2
    3
    4
    5
    解决方案

    答案：不会触发
    ecumator 函数代码周围有一个隐式的try...catch，try...catch只能捕获同步代码中异常。所以 promise 无法处理它。

    # Promise API

    Promise 类有 6 中静态方法。

    # Promise.all

    如果希望并行执行几个 promise，并等待所有 promise 准备就绪。
    例如：并行下载几个 URL，并等待所有内容都下载完毕在进行处理。
    用法：
    下面Promise.all在 3s 后会 settled，返回结果是一个数组['done1', 'done2', 'done3']

    Promise.all([
  new Promise((resolve) => setTimeout(resolve, 1000, "done1")),
  new Promise((resolve) => setTimeout(resolve, 2000, "done2")),
  new Promise((resolve) => setTimeout(resolve, 3000, "done3")),
]).then(alert);
    1
    2
    3
    4
    5

    如果任意一个 promise 被 reject，由Promise.all返回的 promise 就会立刻 reject，并且带回的就是这个 error
    例如：

    Promise.all([
  new Promise((resolve) => setTimeout(resolve, 1000, "done1")),
  new Promise((resolve) =>
    setTimeout(reject, 2000, new Error("Number syntax"))
  ),
  new Promise((resolve) => setTimeout(resolve, 3000, "done3")),
]).catch(alert);
    1
    2
    3
    4
    5
    6
    7

    这里第二个 promise 会在 2s 后 reject，并且导致Promise.all也立刻 reject，也就是说，如果出现 error，其他的 promise 将会被忽略。
